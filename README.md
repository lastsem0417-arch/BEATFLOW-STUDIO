> 
> BeatFlow Studio revolutionizes how creative professionals collaborate. We bridge the gap between Producers, Rappers, Lyricists, and Listeners through an intelligent role-based ecosystem featuring real-time WebRTC collaboration, synchronized digital canvases, intelligent AI assistance, and a thriving global discovery feed.

## 🌟 **Our Vision**

Traditional music production isolates creators. Email attachments break workflows. Time zone differences create chaos. **BeatFlow Studio** shatters these barriers.

We've built a **virtual soundstage** where:
- ✨ Producers and rappers jam together in real-time with crystal-clear video/audio
- 🎼 Lyricists instantly see beat changes and contribute lyrics live
- 🌍 Listeners discover emerging talent and connect with creators
- 📋 Smart contracts document contributions and royalty splits
- 🤖 AI assistance accelerates creativity without replacing it
- 🔐 Encrypted vaults safely archive every creation

## 🚀 **Getting Started**

### **Prerequisites**
- Node.js 16+ and npm/yarn
- MongoDB (local or Atlas)
- Git
- Cloudinary account (for file uploads)
- Google Generative AI API key

### **Installation**

#### **1. Clone Repository**
```bash
git clone https://github.com/YourUsername/beatflow-studio.git
cd beatflow-studio
```

#### **2. Setup Backend**
```bash
cd backend
npm install
```

Create `.env` file in backend directory:
```env
# Server Config
PORT=5000
NODE_ENV=development

# Database
MONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net/beatflow

# Authentication
JWT_SECRET=your_super_secret_jwt_key_here
BCRYPT_ROUNDS=10

# Cloud Storage
CLOUDINARY_CLOUD_NAME=your_cloud_name
CLOUDINARY_API_KEY=your_api_key
CLOUDINARY_API_SECRET=your_api_secret

# AI Services
GOOGLE_AI_API_KEY=your_google_generative_ai_key

# WebRTC (Optional - Uses public STUN/TURN by default)
# TURN_SERVER=your_turn_server
# TURN_USERNAME=your_username
# TURN_PASSWORD=your_password

# CORS & Security
CLIENT_URL=http://localhost:5173
ALLOWED_ORIGINS=http://localhost:5173,http://localhost:3000
```

#### **3. Start Backend**
```bash
npm run dev
```

Backend will run on `http://localhost:5000`

#### **4. Setup Frontend**
```bash
# From root directory
npm install
```

Create `src/Config.js`:
```javascript
const CONFIG = {
  API_BASE_URL: 'http://localhost:5000/api',
  SOCKET_URL: 'http://localhost:5000',
  CLOUDINARY_CLOUD_NAME: 'your_cloud_name'
};

export default CONFIG;
```

#### **5. Start Frontend**
```bash
npm run dev
```

Frontend will run on `http://localhost:5173`

### **Accessing the Application**

1. **Landing Page:** `http://localhost:5173`
2. **Sign Up:** Create account and select role
3. **Dashboard:** Experience role-specific UI
4. **Collaboration:** Join a collab room (test with multiple browser windows)
5. **Admin Panel:** `http://localhost:5173/admin/login`

## 🌐 **Core Modules Breakdown**

### **Authentication & Authorization**
- **JWT-based auth** for secure token management
- **Role-based access control (RBAC):** admin, producer, rapper, lyricist, listener
- **Protected routes** on frontend and backend
- **Session persistence** via sessionStorage

### **Real-Time Communication**
- **Socket.io:** Handles chat, live notifications, and collab room events
- **Room Management:** Isolated event channels for each collaboration session
- **Broadcasting:** System-wide alerts reach all connected users

### **WebRTC Collaboration**
- **Simple-Peer:** P2P video/audio setup with automatic signaling
- **STUN/TURN Servers:** Handle NAT traversal for users behind firewalls
- **Audio Routing:** Separate streams for incoming audio and local microphone
- **Screen Sharing:** Share beats or production setups in real-time

### **File Management**
- **Cloudinary Integration:** Secure cloud storage with CDN delivery
- **Multer middleware:** Handle multipart file uploads
- **Streaming:** Memory-efficient large file handling without buffering entire files

### **AI Assistance**
- **Google Generative AI:** Powered lyrics suggestions, rhyme recommendations, verse expansion
- **Async Processing:** Large requests don't block the server
- **Caching:** Smart caching to reduce API calls

### **Analytics & Insights**
- **Engagement Tracking:** Listen counts, comments, shares
- **Growth Metrics:** Follower trends, reach analysis
- **Creator Matching:** Algorithmic recommendations based on style similarity
